Technical Architecture & Workflow Documentation

1. Technology Stack Breakdown

VeriQ operates on a highly modernized, serverless JavaScript ecosystem optimized for AI speed and modern glassmorphic aesthetics.

Frontend (Client Interface)

  • Next.js 14+ (App Router): Core React framework for seamless page rendering and routing.
  • TypeScript: Type-safe codebase for preventing data mismatch vulnerabilities.
  • Tailwind CSS v4: Core styling engine driving the signature dark mode #070d14 and neon cyan #00d4d4 color scheme.
  • Framer Motion: GPU-accelerated engine powering fluid micro-animations and drop-zone physics.
  • Lucide React: Vector iconography embedded throughout the UI.

Backend & Security Infrastructure

  • Clerk Integration: Zero-trust authentication processing, managing multi-factor candidate login and session security natively via @clerk/nextjs.
  • Next.js Serverless API: High-performance edge/node backend handling secure file buffer processing without a heavy standalone server.
  • pdf-parse (Node.js): Native binary extractor that effortlessly strips raw text syntax directly out of uploaded Resume PDFs.

Artificial Intelligence Core

gemini-pro gemini-1.5-flash @google/generative-ai

The platform natively integrates Google's bleeding-edge Generative AI directly into its pipeline. Gemini is explicitly instructed via systemic prompt engineering to extract strictly structured JSON payloads, drastically reducing output hallucinations.

2. Application System Workflow

This is the seamless, automated user journey from the moment a candidate enters the system to the final analytical scoring report.

1

Authentication Phase

The candidate lands on the VeriQ homepage and selects "Get Started". They are immediately verified through Clerk's SSO overlay, securely logging them into their active candidate profile dashboard.

2

Resume Extraction & AI Target Lock

The candidate drags and drops their technical PDF Resume onto the dashboard. The system uploads the buffer to the /api/parse-resume backend. Node.js strips the text, and Google Gemini aggressively hunts down the explicit core technical skills in the text, formatting them into an array while filtering out padded soft skills.

3

Dynamic Assessment Compilation

The UI visually acknowledges the successfully targeted skills. The candidate clicks "Proceed to Test". Behind the scenes, the /api/generate-test route contacts Gemini to dynamically create 10 bespoke, anti-cheat Technical Multiple Choice Questions (MCQs) mapped precisely to the applicant's declared tech stack.

4

Execution & Scoring Pipeline

The candidate completes the generated exam. The frontend logic securely verifies their choices against the mathematical answers embedded within the AI's hidden JSON payload. The final score is generated, unlocking the capability for high-level Recruiter Dashboards to view precise, validated skill metrics.
